# Archivo documental

Este directorio conserva únicamente specs de features terminadas que aportan trazabilidad SDD. No es una fuente de estado, prioridades ni instrucciones operativas.

Reglas:

- El trabajo activo vive en `specs/`.
- Al cerrar una feature, sus decisiones duraderas pasan a arquitectura/producto y su paquete SDD puede moverse a `docs/archive/specs/`.
- No guardar aquí handoffs, session logs, prompts, resultados crudos, dumps SQL ni copias de documentos canónicos.
- Git ya conserva lo eliminado; no crear un segundo vertedero histórico.
